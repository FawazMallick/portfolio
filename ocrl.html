Overview

This assignment is a Python port of a MATLAB TWIP (two-wheeled inverted pendulum) codebase (twip3.m, main.m, computer_lqr_gains.m, nn_policy.m, nn_cost.m, optimization_script.m, optimize_twip.m). It covers three independent tasks: optimizing linear control gains, learning a neural network policy via imitation learning, and surrogate-based optimization of a 4D benchmark function.

Part 1 Linear Gain Optimization

The TWIP forward dynamics were ported faithfully from twip3.m, a trapezoidal integrator loop matching main.m. A cost function returns the negative survival time when running the linear policy u = −K·x from a small body tilt (x₀ = [0, 0, 0.1, 0]) — minimizing this is equivalent to maximizing how long the robot stays balanced.

Initial gains K = [10, 1, 100, 10]: survival time ≈ 0.006 s (robot falls almost immediately).
After Nelder-Mead optimization (83 function evaluations): gains unchanged, survival time = 0.006 s.
The optimizer confirmed these gains are a local optimum of this cost function for this initial condition.

Part 2 Neural Network Policy via Imitation Learning

A small neural network (4 inputs → 2 hidden tanh units → 1 output, 13 parameters total) was trained to replace the linear controller. Training followed two stages:

Stage 1 — Imitation pre-training: 1,000 random states in [−1, 1]⁴ were generated and labeled with the linear policy's torques. The network was trained to mimic these outputs using L-BFGS-B (MSE loss), achieving a final mimic loss of 0.0002.

Stage 2 — Fine-tuning on survival time: The pre-trained weights were fine-tuned by directly minimizing negative survival time (L-BFGS-B with weight bounds [−10, 10]). The optimized NN policy achieved a survival time of 0.042 s — an improvement over the initial pre-training survival of 0.009 s, but still falling short of robust balancing.

Post-imitation survival: 0.009 s → Post-fine-tuning survival: 0.042 s.
The 13-parameter network is highly constrained, and the survival-time objective is non-smooth and discontinuous, making further improvement difficult with gradient-based methods.

Part 3 Surrogate-Based Optimization — 4D Rosenbrock

The 4D Rosenbrock ("banana") function has a global minimum of 0 at (1,1,1,1). A neural network surrogate (MLP, 10 hidden units) was trained on random samples from the function, then minimized in place of the true function using both constrained (L-BFGS-B, bounds [−2, 2]⁴) and unconstrained (Nelder-Mead) optimization. The experiment was repeated at 100, 500, and 1,000 training samples.

Surrogate quality varies substantially with training set size. The 100-sample surrogates fit the function poorly (true cost at supposed minimum is very high). With 1,000 samples, the surrogate is accurate enough to guide the optimizer close to the true minimum (true cost ≈ 4.0 vs. global minimum of 0). This illustrates the sensitivity of surrogate-based optimization to the quality of the approximating model.
