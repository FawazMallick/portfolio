Overview

This project implements and compares two architectures for Remaining Useful Life (RUL) prediction on the NASA C-MAPSS turbofan engine degradation dataset. A GRU-based baseline is compared against Embed-RUL, a similarity-based method using an encoder-decoder architecture and a learned health index. Both models were trained on the FD001 subset and then evaluated on all four C-MAPSS subsets (FD001–FD004) to assess cross-condition generalization.

Dataset

The NASA C-MAPSS dataset contains run-to-failure trajectories for turbofan engines across four operating conditions and fault modes. Each engine's sensor readings are recorded from initial deployment until failure. FD001 (single operating condition, single fault mode) was used for training and as the primary evaluation benchmark; FD002–FD004 introduce additional operating conditions and fault modes.

Preprocessing used PCA (15 components, retaining 100% of variance on FD001) and MinMaxScaler normalization. Delta features (cycle-to-cycle differences) were also included for the Embed-RUL model.

Architectures

RUL_GRU — GRU Baseline

  • GRU layers followed by a fully connected regression head
  • Trained with an asymmetric scoring loss that penalizes late predictions more heavily than early ones
  • Straightforward sequence-to-scalar prediction

Embed-RUL — Encoder-Decoder with Health Index

  • Encoder (GRUCell) maps engine trajectory to an embedding
  • Decoder reconstructs the trajectory from the embedding
  • A health index (HI) curve is learned that maps RUL onto a [0,1] degradation signal
  • RUL estimation uses similarity between the test engine embedding and training engine embeddings at matching health index values
  • Masking and delta features used to handle variable-length sequences

Results

Both models were trained on FD001 only and evaluated on all four subsets. GRU RMSE is reported for all engines; Embed-RUL reports RMSE only over engines for which a valid prediction could be made (sufficient embedding similarity found).

Embed-RUL produced no valid predictions on FD002 and FD004. The similarity-based retrieval step requires embedding matches within a threshold; when the test set distribution diverges significantly from FD001 training data, no matches are found.

Statistical Comparison (FD001 & FD003)

A paired t-test comparing GRU vs Embed-RUL on FD001 (100 valid engines) and FD003 (65 valid engines) showed that GRU significantly outperforms Embed-RUL on both subsets (p < 0.000001 in both cases).

Visualizations

The project included t-SNE visualizations of encoder embeddings across training and test engines, and health index (HI) curve plots showing the learned degradation signal over engine life. Both help illustrate why the similarity-based approach breaks down under distribution shift.
